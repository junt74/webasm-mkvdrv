# 直近改修計画

この文書は、`README.md` に記載している次の実装ステップを、直近の実作業に落とし込んだメモです。まずは MML パースから AudioWorklet 再生までの最短経路を作り、その後に音源機能を広げます。

## 運用ルール

この文書は固定メモではなく、実装進行に合わせて更新する運用文書として扱います。

ルール:

- 実装を行った都度、この `直近改修計画.md` を更新する
- 完了した項目は完了済みの内容へ反映する
- 優先順が変わった場合は、その理由が読み取れる形で並び順や記述を更新する
- 次に着手する内容が README とこの文書で矛盾しないように保つ
- 大きな方針変更があった場合は、必要に応じて `README.md` 側の「次の実装ステップ」も更新する

## 現在地

現時点で完了している内容:

- Rust/Wasm の開発環境構築
- Vite ベースの `web` 開発環境構築
- サイン波テーブル生成
- Rust 側で生成したノート on/off イベント列の AudioWorklet 再生
- `_reference/mml_spec` への `mmlgui` ベース仕様書追加
- Rust 側の最小 MML パーサ実装
  - 対応済み: `t l o v w @E <> cdefgab r Q q C ^ & R ~ [ ] { }`
  - `:ticks` による tick 直指定を note / rest / length 系で扱える状態まで実装済み
  - `^` による長さ延長と `&` によるレガート化を、現行イベント形式で扱える状態まで実装済み
  - `R` による直前イベント短縮と `~` によるグレースノート追加を、現行イベント形式で扱える状態まで実装済み
  - `[]` は展開型ループとして扱え、`{}` は branch index 文脈を外部から与えて枝選択できる状態まで実装済み
  - パース失敗時に Rust 側から error message と byte offset を取得できる状態まで実装済み
  - Wasm バッファ経由で MML を受け取り、イベント列へ変換できる状態まで実装済み
- Web 側の MML 入力欄追加
  - ブラウザから MML テキストを Rust バッファへ渡して再生できる状態まで実装済み
  - `Play MML` で最小サブセットの再生確認が可能
  - branch selection 用の数値入力と、parse error の位置表示を追加済み
  - parse error 時にエラー位置へフォーカスし、該当位置の選択と周辺行表示ができる状態まで実装済み
  - サンプル MML 切り替えと、入力欄内の簡易ハイライトオーバーレイを追加済み
  - 音符・コマンド・数値・括弧などの軽い色分けと、複数診断表示へ拡張できる描画基盤を追加済み
  - 現在行ハイライトとトークン凡例を追加済み
  - 開き括弧とエラー位置のペアハイライトを追加済み
  - `Sound Engine` 切り替えを追加し、`Sine Test` と `AN74689 PSG` を同じイベント列で切り替え確認できる状態まで実装済み
  - `AN74689` 側の内部状態として `tone 3ch + noise 1ch` を持てる AudioWorklet 構造まで実装済み
  - イベント列へ `channel / volume / noise` 用のメタ情報を載せられる形式へ拡張済み
  - `Start Demo` では `tone 3ch + noise 1ch` を使う PSG デモ列を再生できる状態まで実装済み
  - MML 行頭の `A / B / C / N` 指定で tone 3ch / noise 1ch へイベントを振り分けられる状態まで実装済み
  - MML の各チャンネルイベントを絶対時刻へマージし、`A / B / C / N` を同時再生できる状態まで実装済み
  - `v<num>` を PSG 寄りの coarse volume カーブとして扱える状態まで実装済み
  - `N` チャンネルで `w0 / w1` により periodic / white noise を切り替えられる状態まで実装済み
  - `@E` によるソフトウェアエンベロープ定義と選択を、MML から再生系まで通せる状態まで実装済み
  - `mkvdrv-song` v1 JSON として楽曲データを書き出せるエクスポート API を追加済み
  - `web/src/song-runtime.ts` へ PSG 再生コアを分離し、`web/src/game-audio-engine.ts` から HTML5 ゲーム側で読み込める最小エンジンを追加済み
- Rust 側の診断返却
  - 単一エラー返却に加えて、複数診断を固定長バッファで Wasm 越しに参照できる状態まで実装済み
  - 未対応コマンド候補と loop / conditional 文脈ヒントを診断文面へ付与できる状態まで実装済み
  - 対応する開き括弧位置を関連情報として返し、UI からそこへジャンプできる状態まで実装済み
  - `_reference/mml_spec/commands.md` に結び付いた代替書き方ヒントを返せる状態まで実装済み
  - `_reference/mml_spec/examples.md` に結び付いた短い置き換え断片を返せる状態まで実装済み
  - quick fix 用の置換断片を Wasm 越しに返せる状態まで実装済み
  - Web 側で quick fix の置換前後プレビューを確認してから適用できる状態まで実装済み
  - quick fix プレビューに診断番号、位置、行番号付き差し替え範囲表示を追加済み
  - 1つの診断に対して複数 quick fix 候補を返し、候補ごとにプレビュー切り替えできる状態まで実装済み

次に行う改修は、最小 MML 再生を土台にして、`AN74689` 互換 PSG の土台を実音源らしい period / noise 制御へ近づけつつ、エクスポート形式の圧縮・ループ制御・ゲーム組み込み API を広げる段階へ進めることです。

## 優先順

### 1. `AN74689` 互換 PSG の最小音源コアを育てる

対象:

- `web/src/processor.ts`
- `web/src/song-runtime.ts`
- 必要に応じて `web/src/main.ts`
- 必要に応じて `web/src/game-audio-engine.ts`

内容:

- tone 3ch + noise 1ch の土台を、tone period / noise period / mode の扱いまで寄せる
- 現在の note number 直結を、将来的な PSG period ベースへ移し替えやすい内部表現へ整理する
- coarse volume / noise mode / software envelope を踏まえて、チャンネルごとの差が聴き取りやすいデモを増やす
- ゲーム側 API で BGM / SFX を同居させやすい停止・再開・音量制御を整理する

目的:

- `YM2612` 着手前に、チップ指向の再生エンジン構造を軽量に固める
- MML パーサ出力を PSG らしい発音系へつなぐ

完了条件:

- 複数 PSG チャンネルを独立に発音できる
- noise を含む最小デモをブラウザで確認できる
- periodic / white noise の違いを MML から切り替えて確認できる
- `@E` による音量エンベロープ形状を MML から切り替えて確認できる
- ランタイムが UI から独立し、ゲーム側モジュールから再利用できる

### 2. イベント列フォーマットを拡張する

対象:

- `core/src/lib.rs`
- `web/src/main.ts`
- `web/src/processor.ts`

内容:

- 現在の `note_on / note_off / length_ticks` 形式に加えて、volume / channel / noise / envelope 系の制御を入れられるようにする
- 将来的な PSG / FM 共通化に備えて event kind を整理
- JSON 版 `mkvdrv-song` と将来の compact binary 版で共通に扱える構造へ揃える

目的:

- MML パース結果を無理なく AudioWorklet 側へ渡せるフォーマットにする
- 後で音量、パン、音色変更を追加しやすくする

完了条件:

- Rust 側で複数種別のイベントを直列化できる
- Web 側でイベント種別ごとに分岐して処理できる
- JSON エクスポート結果をゲーム側エンジンで読み込める

### 3. Rust 側の最小 MML パーサを次段へ拡張する

対象:

- `core/src/mml.rs`
- `core/src/lib.rs`

次に対応する内容:

- 将来のループイベント化に向けた内部表現整理
- 将来的な `_{}` や臨時記号の前段となる音高処理整理
- エラー位置情報を byte offset から、必要に応じて line/column や token 種別へ拡張できる形へ整理
- branch selection を単一 index から複数条件文脈へ拡張できる形へ整理

目的:

- `_reference/mml_spec/examples.md` の基本例を段階的に消化する
- 後方修正を伴うコマンドへ進めるための基礎を作る

完了条件:

- Rust テストで length / articulation 系の基本例を検証できる
- 将来の `R`, `~`, `&`, `^` に備えて event builder の編集方針が定まる

## メモ

- `mmlgui` 互換を意識する場合、最初から `_reference/mml_spec/examples.md` の期待値に寄せてテストを書く
- `\` や複雑なループ系は後方修正や分岐が増えるので、今のうちに event builder の責務を崩しすぎない
- イベント列フォーマットは、将来の `volume`, `pan`, `instrument`, `tempo` を見据えて拡張可能に保つ

# 楽曲データエクスポート仕様

この文書は、MML をゲーム実行時依存から切り離し、ビルド時に生成した楽曲データを HTML5 ゲーム側へ読み込ませるための最小仕様をまとめたものです。

## 目的

- ゲーム実行時に MML パースを行わず、軽量な曲データだけを読む
- DCSG 向けの `tempo / note / noise / volume / envelope` を失わずに保存する
- 将来的な `YM2612 / PCM` 追加時にも拡張しやすい JSON ベースの中間フォーマットを用意する

## 形式

- フォーマット名: `mkvdrv-song`
- バージョン: `1`
- 現在の対象エンジン: `sn76489` または `ay38910`
- エクスポート形式: UTF-8 JSON

## ルート構造

```json
{
  "format": "mkvdrv-song",
  "version": 1,
  "engine": "sn76489",
  "ticksPerBeat": 24,
  "channels": [
    { "id": 0, "role": "tone" },
    { "id": 1, "role": "tone" },
    { "id": 2, "role": "tone" },
    { "id": 3, "role": "noise" }
  ],
  "envelopes": [],
  "events": []
}
```

## フィールド

### `format`

- 文字列
- 現状は常に `mkvdrv-song`

### `version`

- 整数
- 現状は `1`

### `engine`

- 文字列
- 現状は `sn76489` または `ay38910`
- 将来 `ym2612` や複合形式を増やす余地を残す

### `ticksPerBeat`

- 整数
- 楽曲全体の tick 基準
- 現在は Rust パーサの `ticks_per_beat` をそのまま出力する

### `channels`

- チャンネル定義配列
- 現状は固定で `tone x3 + noise x1`

| フィールド | 型 | 説明 |
| --- | --- | --- |
| `id` | number | チャンネル番号 |
| `role` | string | `tone` または `noise` |

### `envelopes`

- `@E` 定義を格納する配列

| フィールド | 型 | 説明 |
| --- | --- | --- |
| `id` | number | エンベロープ番号 |
| `speed` | number | 何 tick ごとに次の段へ進むか |
| `values` | number[] | 0..15 の音量段配列 |
| `loopStart` | number? | 末尾到達後に戻る index。未定義なら末尾保持 |

### `pitchEnvelopes`

- `@p` 定義を格納する配列

| フィールド | 型 | 説明 |
| --- | --- | --- |
| `id` | number | ピッチエンベロープ番号 |
| `initialOffset` | number | ノート開始時のピッチオフセット |
| `speed` | number | 何 tick ごとに変化するか |
| `step` | number | 1 回ごとの変化量 |
| `delay` | number | 変化開始まで待つ tick 数 |

### `events`

- 再生用イベント列
- 現在はチャンネルごとの絶対時刻をマージした後の `deltaTicks` 形式

| フィールド | 型 | 説明 |
| --- | --- | --- |
| `deltaTicks` | number | 直前イベントからの差分 tick |
| `kind` | string | イベント種別 |
| `channel` | number | 対象チャンネル |
| `value` | number | 主値 |
| `param` | number | 補助値 |

## `kind` 一覧

| `kind` | 意味 | `value` | `param` |
| --- | --- | --- | --- |
| `tempo` | テンポ変更 | BPM | 0 |
| `volume` | coarse volume 設定 | 0..15 | noise の場合は mode |
| `noteOn` | tone ノート開始 | note number | volume |
| `noteOff` | tone ノート終了 | note number または 0 | 0 |
| `noiseOn` | noise 開始 | 擬似周波数 | 下位 8bit volume / 上位 8bit mode |
| `noiseOff` | noise 停止 | 擬似周波数または 0 | 0 |

補足:

- `noiseOn.value` はエクスポート上は擬似周波数のまま保持します
- `SN76489` ランタイムでは、この値を近い離散 noise source へ量子化して再生します
- `AY-3-8910` ランタイムでは、現状は連続寄りの noise 周波数として扱います
| `envelopeSelect` | `@E` 選択 | envelope id | 0 |
| `pan` | パン設定 | `0=OFF`, `1=右`, `2=左`, `3=左右` | 0 |
| `pitchEnvelopeSelect` | `@p` 選択 | pitch envelope id | 0 |

## 現行ランタイムでの解釈

- `web/src/game-audio-engine.ts` が HTML5 ゲーム側の最小導線
- `web/src/song-format.ts` が JSON スキーマ対応型と変換関数
- `web/src/song-runtime.ts` が UI 依存を持たない PSG 再生コア
- `web/src/processor.ts` は AudioWorklet メッセージ層だけを担当する

## 実装上の方針

- まずは可視性の高い JSON を正式な開発用形式とする
- サイズ最適化が必要になった段階で、同じ意味を持つバイナリ版を追加する
- バージョン差異は `format + version` で判定する
- 互換性を壊す変更は `version` を上げる

## 次段の拡張候補

- 曲メタデータ: `title`, `author`, `loop`, `comment`
- ループ再生位置の明示
- `YM2612 / PCM` 用イベント拡張
- 効果音向けの短尺データ最適化
- JSON と等価な compact binary 形式

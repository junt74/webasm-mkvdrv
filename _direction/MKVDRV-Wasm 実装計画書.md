# **MKVDRV-Wasm 実装計画書 (V1.2)**

## **1\. プロジェクト概要**

本プロジェクトは、HTML5ゲームにおけるBGMおよびSE再生を主目的とし、MKVDRV の設計思想とコード資産をベースに、JavaScript/WebAssembly（Wasm）環境で動作する次世代サウンドドライバを構築する。実機を超える「クリスタルクリア」な音質と、ブラウザ上での「爆速な楽曲制作イテレーション」の両立、および **MDSDRV-mml との完全な上位互換性** の確保を目指す。

## **2\. システムアーキテクチャ**

ブラウザのメインスレッドと音声専用スレッド（AudioWorklet）を分離し、処理の安定性と低遅延を両立する。

* **Main Thread (UI/Editor/Parser):**  
  * MML エディタ（Monaco Editor 等）による編集環境の提供。  
  * MML のランタイムパース（Rust/Wasm）。  
  * アセット（PCM/音色）の管理と AudioWorklet への転送。  
* **AudioWorklet Thread (Synthesis Engine):**  
  * サンプル精度（Sample-accurate）のシーケンサ。  
  * 音源エミュレーション（YM2612 / SN76489 / 4ch PCM）。  
  * 高品質リサンプリングおよび DAC 出力。  
* **通信:** SharedArrayBuffer を用いたロックフリーな双方向通信プロトコル。

## **3\. 音源部（Sound Core）仕様**

### **3.1 エミュレーション・コア**

* **YM2612 (OPN2):** 6ch FM 音源。MAME 等の OSS 資産を Rust へ移植し、内部演算を浮動小数点化。  
* **SN76489 (PSG):** 3ch PSG。正確なノイズ生成アルゴリズムを維持。  
* **カスタム PCM (4ch 拡張仕様):**  
  * **互換モード:** 従来の MDSDRV 互換（1ch DAC）として動作しつつ、残り 3ch を独立した拡張スロットとして運用可能。  
  * **機能:** 任意レート・音程付き再生、高品質な補間アルゴリズム（Sinc / Cubic）を搭載。  
* **DAC 出力:**  
  * 32bit Float パイプラインによる「クリスタルクリア」な音質。  
  * ユーザー指定の出力周波数（48k / 44.1k / 22.05k）に対応。  
  * 内部合成レートとデバイスレート間の高品質な帯域制限リサンプリングを Rust で実装。

### **3.2 優先順位制御（Priority Manager）**

* BGM と SE を同一フォーマットで扱い、チャンネルごとに「BGM 優先」「SE 優先」をフラグで設定可能。  
* SE 再生によって一時的に奪われた BGM チャンネルは、SE 終了後に元の音色・音高へ自動復帰するスタック構造。

## **4\. 楽曲データ処理（MML & Sequencer）**

### **4.1 MML 仕様（MDSDRV-mml 上位互換）**

* **既存コマンドの完全サポート:** \#WAV 定義、% による発音コマンド等、従来の MML を無加工で再生可能。  
* **拡張シンタックス:** % コマンドへの追加引数による 4ch スロット指定、ピッチベンド、エンベロープ適用等。  
* **ランタイムパース:** ブラウザ実行時にテキストをバイナリ命令列（Bytecode）へ即時変換。

### **4.2 シーケンシング**

* **サンプル精度:** AudioWorklet の currentFrame を唯一の絶対時間（オーディオクロック）として採用。  
* **ドリフト対策:** システムクロックに依存せず、処理済みサンプル数に基づいて正確にカウントダウンを行う。  
* **ライブリロード:** 再生を中断せず、演奏命令列のみをアトミックに差し替えるホットスワップ機構。

## **5\. アセット管理（PCM & Memory）**

### **5.1 ハイブリッド管理モデル**

* **開発時 (Editor Mode):** エディタへのドラッグ＆ドロップで PCM スロットを即時更新。  
* **本番時 (Production Mode):** シーンに必要なアセットを 1 つのバイナリパッケージとしてプリロード。

### **5.2 動的パッキング方式**

* **メモリ配置:** シーンごとに必要な PCM データのみを Wasm 線形メモリへ隙間なくパッキング。  
* **メモリアライメント:** SIMD 最適化のため、各データを 16 バイト境界で整列。  
* **アドレステーブル:** 従来の \#WAV \[ID\] を Wasm 側のアドレステーブルにマッピングし、高速なランダムアクセスを実現。

## **6\. 実装環境の構築**

### **6.1 プロジェクト構造 (Monorepo)**

/MKVDRV-wasm  
├── /core (Rust: Wasm Source)  
│   ├── Cargo.toml  
│   ├── build.rs (native内のCコードを自動ビルド)  
│   ├── /native (OSS資産のコード本体: ビルド対象)  
│   │   ├── /mame (YM2612等のコア)  
│   │   └── /nuked (Nuked-OPN2等)  
│   └── /src (Rust Wrapper, Mixer, Sequencer)  
├── /web (TS/Vite: Frontend)  
│   ├── /src/processor.ts (AudioWorklet本体)  
│   └── /src/editor (Monaco setup)  
├── /\_reference (AI参照用: ビルド対象外)  
│   ├── MDSDRV\_Original\_Source (オリジナルのMML解釈ロジック等)  
│   └── mmlgui\_specs (文法仕様書等)  
├── /assets (PCM, MML samples)  
└── .cursorrules (AI への指示書)

### **6.2 資産管理の使い分け**

1. **/core/native:** build.rs から直接参照し、Wasm へコンパイルする対象。最小限の .c / .h ファイルを配置。  
2. **/\_reference:** AI（Cursor 等）へのコンテキスト提供用。オリジナルコードの意図を解析させるために使用。

### **6.3 AI 駆動開発のディレクション**

* **FFI ブリッジ:** C 関数のシグネチャを AI に読み込ませ、Rust の extern "C" 定義と安全なラッパーを自動生成。  
* **リファクタリング:** \_reference 内のロジックを解析し、現代的な Safe Rust での再実装を AI に指示。

## **7\. 開発ロードマップ**

* **Stage 1:** Rust \+ AudioWorklet の最小構成で音出し（サイン波）。  
* **Stage 2:** YM2612（C 資産）のリンクと、サンプル精度シーケンサの基本実装。  
* **Stage 3:** MKVDRV-mml パーサの実装（既存コマンドの完全解析）。  
* **Stage 4:** 動的パッキング PCM エンジンと優先順位マネージャの統合。  
* **Stage 5:** Monaco Editor を統合したライブ編集・プレビュー環境の構築。

# MML コマンド仕様

この文書は `/Users/junt74/Projects/cpp/mmlgui/ctrmml` の実装を元に、MKVDRV-Wasm で引き継ぐべき MML コマンド仕様を整理したものです。主な参照元は以下です。

- `ctrmml/src/mml_input.cpp`
- `ctrmml/src/track.cpp`
- `ctrmml/src/track.h`
- `ctrmml/src/unittest/test_mml_input.cpp`
- `ctrmml/mml_ref.md`

ここでは「実装で実際に解釈されている挙動」を優先しています。説明の一部は実装からの推論を含みます。

## 行構造

各行は次のいずれかとして解釈されます。

- トラック指定行
- タグ行 (`#` または `@` で開始)
- コメント行 (`;` で開始)
- 空行または継続行

トラック指定を省略した行は、直前に選択されていたトラック群を引き継ぎます。

例:

```text
A    cdef
     bagf ; どちらも track A に追記される
```

## トラック指定

行頭に空白なしで記述された文字列がトラック選択子になります。

- `A` から `Z` はトラック `0` から `25`
- `0` から `9` はトラック `26` 以降
- `*<num>` は明示的なトラック番号

複数トラックを連続して書くと、その行の MML は全トラックに複製されます。

例:

- `ABC cdef`
- `*10 l8 cdefga`

## タグ・メタコマンド

### `#...`

`#` で始まるタグは単一文字列として扱われます。

- `#platform` は特別扱いで、`Song::set_platform()` に渡される
- それ以外は `Song::set_tag()` で置換保存される

代表例:

- `#title`
- `#composer`
- `#author`
- `#date`
- `#comment`

### `@...`

`@` で始まるタグは値リストとして扱われ、継続行で追記されます。

- スペースまたはカンマ区切り
- ダブルクォートで囲んだ文字列は 1 要素として扱う
- 直後のインデント行は同じタグへ追加される

用途:

- `@<num>` 音色定義
- `@E<num>` 音量エンベロープ
- `@M<num>` ピッチエンベロープ
- `@P<num>` パンエンベロープ

補足:

- MKVDRV-Wasm では `@E` をカスタム定義エンベロープとして扱う
- 一方 `S<num>` は役割固定のプリセットエンベロープ選択に使う
- `S` の番号意味はチップモデルをまたいで共通にし、実カーブのみを各チップ向けに最適化する方針
- AY hardware envelope は `EH` `EP` `EE` の 2 文字コマンドで分離する

## 基本ノート系コマンド

### 音符 `c d e f g a b h`

`a` から `h` が音符です。`h` は `b` と同義です。

実装上の音名テーブルは次の通りです。

- `c = 0`
- `d = 2`
- `e = 4`
- `f = 5`
- `g = 7`
- `a = 9`
- `b = 11`
- `h = 11`

ドラムモードでない場合は、ここにオクターブ値とキーシグネチャが加算されます。

### 休符 `r`

指定長の休符を挿入します。

### タイ `^`

前のノートまたは休符の長さを延長します。

- 直前イベントが `NOTE` または `REST` の場合はその長さを延長
- 途中に別イベントが挟まっている場合は `TIE` または `REST` が追加される

### スラー `&`

前のノートをレガート化します。

- 直前の `NOTE` または `TIE` の `off_time` を `0` にし、`on_time` に吸収
- 合わせて `SLUR` イベントを追加
- 直前に有効なノートがない場合は警告相当

### グレースノート `~<note><duration>`

直前ノートから長さを借りて、短い前打音を挿入します。

処理順:

1. `reverse_rest(duration)` で前イベントを短縮
2. 指定音長のノートを追加

### リバースレスト `R<duration>`

直前ノートまたは休符の長さを短縮します。

- まず `off_time` から減算
- 足りなければ `on_time` から減算
- ループ末尾やセグノをまたいだ逆算はエラー

## 音高とオクターブ

### `o<num>`

オクターブ設定です。実装では `set_octave(expect_parameter() - 1)` のため、MML 上の `o4` は内部オクターブ `3` になります。

テストから確認できる例:

- `o4 c` はノート番号 `36`
- `o4 d` は `38`

### `<` と `>`

- `<` はオクターブを 1 下げる
- `>` はオクターブを 1 上げる

## 長さ指定

### 直接長

音符や休符の後ろに数値を付けると、その長さを「全音符長 / 分母」で計算します。

例:

- `l16 c` は 16 分音符
- `d8` は 8 分音符

### フレーム直接指定 `:<ticks>`

長さの先頭に `:` を付けると、分母ではなく tick 値を直接指定します。

例:

- `c:24`

### 付点 `.`

付点は繰り返し指定可能で、都度「前回加算値の半分」を足します。

例:

- `8.` は `1/8 + 1/16`
- `8..` は `1/8 + 1/16 + 1/32`

### `l<duration>`

デフォルト長を設定します。以後、個別指定のない音符・休符・タイで使われます。

### `C<ticks>`

全音符長を tick 単位で設定します。デフォルトは `96` です。

例:

- `C96 c4` は 24 tick
- `C128 c4` は 32 tick

## 発音長・アーティキュレーション

### `Q<1..8>`

量子化比率を設定します。`on_time = duration * Q / 8` の意味になります。

- `Q0` は `Q8` と同義
- `Q` 設定時は early release をクリア

### `q<ticks>`

早めのキーオフを tick 単位で設定します。

- `on_time = duration - q`
- ただし `q >= duration` の場合は `on_time = duration - 1`
- `q` 設定時は quantize を実質 `8/8` に戻す

## ループ・制御構文

### `[` と `]`

ループブロックです。

- `[` は `LOOP_START`
- `]` は `LOOP_END`
- `]` の省略時デフォルト回数は `2`

例:

- `[c]`
- `[c]5`

### `/`

ループの最終回だけスキップされる break 節です。`LOOP_BREAK` を追加します。

### `L`

セグノです。トラック終端到達時の戻り位置を表す `SEGNO` を追加します。

### `*<num>`

別トラックの内容をサブルーチン的に実行する `JUMP` です。

注記:

- トラック選択子の `*<num>` と、MML 本文中の `*<num>` は別用途

### `{.../.../...}`

複数トラック選択時の条件分岐ブロックです。

- トラックリスト中の何番目かで枝を選ぶ
- `/` で枝を区切る
- `}` で閉じる
- 未閉鎖ならエラー

例:

```text
ABC o4c{d/e/f}g
```

この場合、A は `d`、B は `e`、C は `f` が入ります。

## エコー

### `\=<delay>,<volume>`

エコー設定です。

- `delay` は何個前のノートを参照するか
- 正の値なら echo buffer をクリア
- 負の値なら絶対値を使い、buffer は保持
- `volume` は相対音量変化量

### `\<duration>`

エコーノートを追加します。

- 対象がノートならそのノートを再発音
- 対象が休符または buffer 不足なら休符化
- 音量差分があれば前後に `VOL_REL` を挿入

## トランスポーズ・キーシグネチャ

### `_<num>`

絶対トランスポーズです。`TRANSPOSE` イベントになります。

### `__<num>`

相対トランスポーズです。`TRANSPOSE_REL` イベントになります。

### `_{...}`

キーシグネチャ設定です。

2 種類あります。

- 調名指定: `_{C}`, `_{a}`, `_{c+}`, `_{B-}` など
- 修飾指定: `_{+cfg}`, `_{-be}`, `_{=dg}` など

調名指定は既存キーシグネチャをリセットし、修飾指定は現在設定へ差分適用します。

## 音量・音色・制御イベント

### `@<num>`

音色番号指定。`INS`

### `v<num>`

粗い音量。`VOL`

### `(` `)`

相対音量変更。デフォルト変化量は `1`

- `(` は `VOL_REL` の負方向
- `)` は `VOL_REL` の正方向

### `V<num>` / `V+<num>` / `V-<num>`

細かい音量です。

- 符号なしは `VOL_FINE`
- `+` / `-` 付きは `VOL_FINE_REL`

### `p<num>`

パン。`PAN`

### `k<num>` / `K<num>`

- `k` は現実装では `_` と同じ扱い
- `K` は detune。`DETUNE`

### `E<num>` `M<num>` `P<num>` `G<num>`

それぞれ次のイベントになります。

- `E` → `VOL_ENVELOPE`
- `M` → `PITCH_ENVELOPE`
- `P` → `PAN_ENVELOPE`
- `G` → `PORTAMENTO`

### `D<num>`

ドラムモードです。

- `0` で無効
- 非 0 で有効化し、ノート番号に基底値として加算
- ドラムモード中は通常の音名変換とキーシグネチャ適用を行わない

### `t<num>` / `T<num>`

- `t` は BPM ベース。`TEMPO_BPM`
- `T` はプラットフォーム依存テンポ。`TEMPO`

## プラットフォーム固有

### `'<text ...>'`

シングルクォートで囲まれたメッセージはプラットフォーム固有コマンドです。

- `Song::register_platform_command()` に登録
- トラックには `PLATFORM` イベントが積まれる
- `param` には内部コマンド ID が入る

### `%<num>`

本文中の `%` は現実装では `PLATFORM` イベントとして整数パラメータを直接積みます。

これは `mml_ref.md` でいう「direct length」ではなく、`ctrmml` 実装で独自に割り当てられた処理です。MKVDRV-Wasm 側で継承するかどうかは別途判断が必要です。

## コメントと区切り

- `;` 以降は行末までコメント
- `|` は視覚区切りで、実行上は無視される

## エラー条件

実装・テストから確認できる代表例:

- `l0`, `l-2`, `l:-5` は不正長
- 未閉鎖の条件ブロックはエラー
- 未閉鎖のプラットフォームメッセージはエラー
- `R` や `~` で前イベントを十分に短縮できない場合はエラー

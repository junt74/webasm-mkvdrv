# SN76489 実装メモ

## まず揃えるもの

- tone 3ch の period ベース内部表現
- noise channel の period と mode
- 4bit volume の減衰カーブ
- チャンネル状態をレジスタ寄りに扱える更新順

## 先にやる理由

- 現在の `MKVDRV-Wasm` は `tone 3ch + noise 1ch` の骨格がすでにあり、`SN76489` 側を基準モデルにしやすい
- `AY-3-8910` との差分を後から積む際に、共通 PSG の責務を整理しやすい

## 直近の実装テーマ

1. tone / noise / volume を register shadow として保持する
2. noise 周波数と LFSR 更新を `SN76489` コアへ寄せる
3. latch/data write 相当の更新順を表現する
4. ブラウザ確認用に、tone/noise 差が分かりやすい比較サンプルを増やす

## 現在の到達点

- tone channel は `period register` と `volume register` を shadow state として保持する段階まで到達
- noise channel は `noise control register` と `volume register` を shadow state として保持する段階まで到達
- chip core 側に `tone period / volume / noise control` を書き込む helper を追加済み
- まだ実クロック由来 counter や LFSR 更新順の完全分離までは持っていない

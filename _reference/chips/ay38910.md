# AY-3-8910 実装メモ

## 現在の位置づけ

現状の `AY-3-8910` は、出音差を比較するための簡易モデルです。振幅カーブや noise の違いは一部入っていますが、mixer / hardware envelope / register 単位の厳密さはこれからです。

## 段階的な導入順

1. `SN76489` と同じ共通 PSG 骨格の上で、AY 用コアを分離する
2. AY の振幅テーブル差分を独立責務にする
3. tone/noise mixer を導入する
4. hardware envelope を導入する
5. 必要なら I/O port 相当は「非対象」と明記する

## 実装観点

- AY は tone と noise の有効/無効を mixer でチャンネルごとに切り替える設計なので、現在の単純 pan とは責務が異なる
- volume は単純な線形ではなく、段差の感じ方が `SN76489` と異なる
- hardware envelope を導入する場合、現行 `@E` との責務分離を設計してから追加した方が安全
- hardware envelope の shape / period / enable は `EH / EP / EE` として暫定導入済み
- shape の bit 解釈と `EH0` から `EH15` の整理は `_reference/mml_spec/ay_hardware_envelope.md` を基準にする

## hardware envelope 厳密化の観点

- `EH` 4bit を `continue / attack / alternate / hold` として解釈する
- `EH9`, `EH10`, `EH13`, `EH14` を代表 shape として耳確認しやすい
- 現行実装は最小 generator であり、終端処理や反転タイミングは今後の厳密化対象
- software envelope と hardware envelope は当面排他で扱う
- AY mixer 実装時に、hardware envelope の出力をどの段階で volume に結び付けるかを再確認する

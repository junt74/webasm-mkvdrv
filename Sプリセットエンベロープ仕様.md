# S プリセットエンベロープ仕様

この文書は、`S<num>` コマンドで選択できる PSG 向けプリセットエンベロープの暫定仕様をまとめたものです。

`@E` がユーザー定義エンベロープであるのに対し、`S` は「すぐ鳴らして試せる内蔵プリセット」の位置付けです。MSX BASIC 系 MML の `S` を参考にしつつ、現行 MKVDRV-Wasm の PSG 実装に合わせて最小限の種類から始めます。

今後は、`S1` から `S9` の番号ごとの意味を `SN76489` / `AY-3-8910` で共通に保ちます。どのチップでも `S3` は「速い立ち上がり」、`S6` は「長めの減衰」という役割を持ち、実際のカーブ値だけを各チップモデルの音量段階や出音差に合わせて最適化します。

## 基本仕様

- 書式は `S<num>` です
- `S0` はプリセット解除です
- `S1` 以上は内蔵プリセットを選択します
- 番号ごとの役割はチップモデルをまたいで共通です
- 実際の音量カーブはチップごとに最適化される場合があります
- 実行時には内部的に予約済みエンベロープ ID へ変換され、楽曲データへも通常の envelope 定義として出力されます
- ノート Key-On ごとに先頭から再生されます
- 値の意味は既存 `@E` と同じで、`0` が最大、`15` が消音です

## プリセット一覧

| S | 名前 | ねらい | 展開イメージ |
| --- | --- | --- | --- |
| `S0` | off | エンベロープ解除 | なし |
| `S1` | gate-full | 発音中ほぼ一定音量の単純ゲート | `{1,0}` |
| `S2` | gate-soft | 一定寄りだが少し抑えたゲート | `{1,4}` |
| `S3` | attack-fast | 速い立ち上がり | `{1,12,8,4,0}` |
| `S4` | attack-slow | 緩やかな立ち上がり | `{2,14,12,10,8,6,4,2,0}` |
| `S5` | decay-short | 短く減衰する打撃音向け | `{1,0,4,8,12,15}` |
| `S6` | decay-long | 長めに減衰する標準形 | `{2,0,2,4,6,8,10,12,14,15}` |
| `S7` | decay-hold | 減衰後に浅く保持感を残す | `{1,0,3,6,8,8,8,8}` |
| `S8` | dip-return-fast | 一瞬だけへこんですぐ戻る | `{1,0,8,0}` |
| `S9` | dip-return-wide | もう少し深くへこんで戻る | `{1,0,10,4,0}` |

## 使い分けの目安

- 単純ゲートで十分な矩形波らしい鳴り方なら `S1`
- 少し引っ込めた伴奏には `S2`
- ピコッと立ち上がる効果音や短音には `S3`
- ふわっと入る音色には `S4`
- 打鍵感のある減衰には `S5`
- 長めに尾を引かせたいときは `S6`
- 減衰後も少し芯を残したいときは `S7`
- アタックに一瞬のへこみを入れたいときは `S8` / `S9`

## チップ間の扱い

- `S` の番号意味は `SN76489` と `AY-3-8910` で共通です
- 同じ `S6` でも、内部の段数や値列は各チップの音量特性に合わせて別実装で構いません
- MML 上では「同じ番号なら同じ役割」と理解できる状態を優先します
- `@E` は詳細なカスタム定義、`S` は意味固定の簡易プリセット、という役割分担を維持します

## 例

```mml
A t132 o4 l8 S3 c e g > c
B o3 l8 S5 c c g g
N l8 S8 w1 c r c r
```

同じフレーズでも `S1`, `S3`, `S5` を切り替えると、ゲート感・立ち上がり・減衰の印象差をすぐ比較できます。より細かく追い込みたい場合は、最終的に `@E` 定義へ移行してください。

# MKVDRV-Wasm ユーザーマニュアル

この文書は、現在実装済みの DCSG 向け機能を使って、

- MML から楽曲データをエクスポートする
- HTML5 コンテンツへ再生エンジンを組み込む
- ゲーム側から `play()` を呼んで再生する

までを一連の流れとしてまとめたユーザー向け手順書です。

対象は現時点の `SN76489` / `AY-3-8910` 切り替え対応 PSG ベース実装です。

## 1. できること

現状の MKVDRV-Wasm では、次の流れを取れます。

1. MML を Rust/Wasm 側で解析する
2. `mkvdrv-song` v1 JSON として楽曲データを書き出す
3. HTML5 側でその JSON を読み込む
4. `MkvdrvGameAudioEngine` を使って再生する

対応している楽曲要素の例:

- tone 3ch + noise 1ch
- テンポ
- coarse volume
- パン (`p0` から `p3`)
- noise mode (`w0`, `w1`)
- ソフトウェアエンベロープ (`@E`)
- ピッチエンベロープ (`@p`)
- プリセットエンベロープ (`S`)
- チャンネル分割 (`A`, `B`, `C`, `N`)

## 2. 前提環境

必要なもの:

- Rust stable
- `wasm32-unknown-unknown` target
- Node.js / npm

開発時の基本コマンド:

```bash
cargo test -p mkvdrv-wasm-core
cd web
npm install
npm run dev
```

## 3. MML を用意する

まずは通常どおり MML を作成します。

例:

```mml
@E1={1,0,2,4,6,8,10,12,14}
@E2={2,0,4,8,12,255,1}

A t132 o4 l8 S3 v14 c e g > c
B o3 l8 @E2 v11 c c g g
N l8 S8 v9 w1 c r c r
```

ポイント:

- `@E1={...}` のようにエンベロープを定義できます
- `@E1={speed, level1, level2, ...}` の先頭要素は進行速度です
- `speed=1` はほぼ毎ティック進み、`speed=2` は 2 ティックごとに 1 段進みます
- `@E1` のようにトラック内でエンベロープを選択できます
- `@E` の音量値は `0` が最大、`15` が最小です
- つまり `@E` の要素数字は大きいほど音量が小さくなります
- `@p1={initial, speed, step, delay}` でピッチエンベロープを定義できます
- `@p1` でトラック内へピッチエンベロープを選択できます
- `@p` は `100 = 半音 1 つ分` として扱います
- `S1` から `S9` で、内蔵プリセットエンベロープをすぐ選べます
- `S0` でプリセット選択を解除できます
- `A / B / C / N` で tone/noise チャンネルへ振り分けます
- `p0=OFF`, `p1=右`, `p2=左`, `p3=左右` でチャンネルごとのパンを指定できます
- `N` チャンネルでは `w0` が periodic noise、`w1` が white noise です
- `SN76489` モデル時は、`N` チャンネルの音高は近い離散 noise source に寄せて再生します
- プリセット一覧は [Sプリセットエンベロープ仕様.md](/Users/junt74/Projects/webasm/webasm-mkvdrv/Sプリセットエンベロープ仕様.md) を参照してください
- `@p` の詳細は [ピッチエンベロープ仕様.md](/Users/junt74/Projects/webasm/webasm-mkvdrv/ピッチエンベロープ仕様.md) を参照してください

## 4. 楽曲データをエクスポートする

### ブラウザ UI から書き出す方法

1. `cd web`
2. `npm run dev`
3. ブラウザでページを開く
4. `MML Input` に楽曲を入力する
5. 必要なら `Branch Select`, `Renderer`, `Chip Model` を調整する
6. `Export Song JSON` を押す

成功すると `mkvdrv-song.json` が保存されます。

この JSON は、現在の最小ゲーム向け交換形式です。

### 出力形式

出力は `mkvdrv-song` v1 JSON です。

ルート構造の例:

```json
{
  "format": "mkvdrv-song",
  "version": 1,
  "engine": "sn76489",
  "ticksPerBeat": 24,
  "channels": [
    { "id": 0, "role": "tone" },
    { "id": 1, "role": "tone" },
    { "id": 2, "role": "tone" },
    { "id": 3, "role": "noise" }
  ],
  "envelopes": [],
  "events": []
}
```

詳細仕様は [楽曲データエクスポート仕様.md](/Users/junt74/Projects/webasm/webasm-mkvdrv/楽曲データエクスポート仕様.md) を参照してください。

## 5. HTML5 コンテンツへ組み込む

ゲーム側で使う最小導線は [web/src/game-audio-engine.ts](/Users/junt74/Projects/webasm/webasm-mkvdrv/web/src/game-audio-engine.ts) です。

内部構成は次のようになっています。

- [web/src/song-format.ts](/Users/junt74/Projects/webasm/webasm-mkvdrv/web/src/song-format.ts)
  `mkvdrv-song` JSON の型と再生用 payload 変換
- [web/src/song-runtime.ts](/Users/junt74/Projects/webasm/webasm-mkvdrv/web/src/song-runtime.ts)
  UI 非依存の PSG 再生コア
- [web/src/processor.ts](/Users/junt74/Projects/webasm/webasm-mkvdrv/web/src/processor.ts)
  AudioWorklet の受け口
- [web/src/game-audio-engine.ts](/Users/junt74/Projects/webasm/webasm-mkvdrv/web/src/game-audio-engine.ts)
  HTML5 ゲーム側から扱うための API

### 最小手順

ゲーム側では次のものを用意します。

1. エクスポート済み `mkvdrv-song.json`
2. `MkvdrvGameAudioEngine` を含む配布用 JS

`wavetable` と `note frequency table` は、現状の `MkvdrvGameAudioEngine` が内部で生成するため、ゲーム側で別途用意する必要はありません。

## 6. 最小コード例

以下は、エクスポート済み JSON を読み込んで `play()` する最小イメージです。

```ts
import { MkvdrvGameAudioEngine } from "./mkvdrv-game-audio-engine.js";
import type { ExportedSong } from "./song-format";

const response = await fetch("/audio/mkvdrv-song.json");
const song = (await response.json()) as ExportedSong;

const engine = new MkvdrvGameAudioEngine({
  initialFrequency: 440
});

await engine.initialize();
await engine.loadSong(song);
await engine.play();
```

### URL から直接読み込む最小例

```ts
import { MkvdrvGameAudioEngine } from "./mkvdrv-game-audio-engine.js";

const engine = new MkvdrvGameAudioEngine();

await engine.initialize();
await engine.loadSongFromUrl("/audio/mkvdrv-song.json");
await engine.play();
```

## 7. 配布用 JS を作る

ゲーム側へ渡すエンジン JS は、`web` ディレクトリで次のコマンドから生成できます。

```bash
cd web
npm run build:engine
```

生成物は `web/dist-engine/` に出力されます。

主に使うもの:

- `mkvdrv-game-audio-engine.js`
- `mkvdrv-processor.worklet.js`
- `wasm/mkvdrv_wasm_core.wasm`

ゲーム側へ組み込むときは、`dist-engine/` の出力物をまとめて配置してください。

役割の分け方:

- `mkvdrv-game-audio-engine.js`
  ゲーム側から直接呼ぶ公開 API
- `mkvdrv-processor.worklet.js`
  AudioWorklet 専用の内部実装
- `wasm/mkvdrv_wasm_core.wasm`
  再生コアが参照する Wasm バイナリ

`mkvdrv-game-audio-engine.js` 単体では動作しないため、3 点を同じ相対構成のまま配布する必要があります。

配置イメージ:

```text
public/
  audio/
    stage1-bgm.json
  engine/
    mkvdrv-game-audio-engine.js
    mkvdrv-processor.worklet.js
    wasm/
      mkvdrv_wasm_core.wasm
```

## 8. `play()` までの標準手順

ユーザー操作を起点にする場合は、次の順番が分かりやすいです。

```ts
button.addEventListener("click", async () => {
  await engine.initialize();
  await engine.loadSong(song);
  await engine.play();
});
```

ブラウザの AudioContext 制約があるため、初回再生はクリックやタップなどのユーザー操作から始めるのが安全です。

## 9. 利用できる API

`MkvdrvGameAudioEngine` では、現時点で次の API が使えます。

### `initialize()`

- `AudioContext` と `AudioWorkletNode` を初期化します
- 初回に一度呼べば十分です

### `loadSong(song)`

- `mkvdrv-song` JSON をエンジンへ設定します
- 再生前に呼びます

### `loadSongFromUrl(url)`

- JSON ファイルを `fetch()` で読み込み、そのままロードします
- 静的配置した `mkvdrv-song.json` を扱う場合に便利です

### `play()`

- 現在ロード済みの曲を再生します
- 曲が未ロードだと例外になります

### `stop()`

- 再生停止し、AudioContext を suspend します

### `pause()`

- AudioContext を suspend します

### `resume()`

- pause 後の再開に使います

### `setMasterVolume(volume)`

- 0.0 から 1.0 の範囲で全体音量を設定します

例:

```ts
engine.setMasterVolume(0.5);
```

### `setRenderEngine(renderEngine)`

- `engine` には `sn76489` または `ay38910` が入ります
- Renderer は `psg` または `sine` を切り替えます
- 実運用は `psg` を前提にし、`sine` は確認用と考えるのが自然です

## 10. 実装例

### BGM の開始

```ts
await engine.loadSong(song);
await engine.play();
```

### 一時停止と再開

```ts
await engine.pause();
await engine.resume();
```

### 停止

```ts
await engine.stop();
```

### 音量変更

```ts
engine.setMasterVolume(0.75);
```

## 11. JSON ファイルの配置例

例えば Vite や静的ホスティングなら、次のような配置が扱いやすいです。

```text
public/
  audio/
    stage1-bgm.json
```

ロード例:

```ts
const song = (await fetch("/audio/stage1-bgm.json").then((res) => res.json())) as ExportedSong;
```

## 12. 注意点

- 現在のエクスポート形式は開発用 JSON が基準です
- 実行時に MML を読む設計ではなく、事前エクスポートして使う前提です
- `play()` 前には `loadSong()` が必要です
- 初回再生はユーザー操作起点にしてください
- `MkvdrvGameAudioEngine` は内部で wavetable と note frequency table を生成します
- ゲーム側へは `dist-engine/` を丸ごと、または同じ相対構成のまま配置してください
- 現時点では loop 専用 API や SFX ミキシング専用 API は未実装です

## 13. 現状のおすすめ運用

最初は次の運用がいちばん安全です。

1. 開発中は `web` UI で MML を調整する
2. `Play MML` で音を確認する
3. `Export Song JSON` で `mkvdrv-song.json` を出力する
4. ゲーム側で JSON を `fetch()` する
5. `MkvdrvGameAudioEngine` に `loadSong()` して `play()` する

この流れなら、作曲・確認・ゲーム組み込みの責務が分かれ、後で binary 化や FM 対応へ進めるときも移行しやすくなります。

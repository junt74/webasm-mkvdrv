# ピッチエンベロープ仕様

この文書は、現行 MKVDRV-Wasm における `@p` ピッチエンベロープの最小仕様をまとめたものです。

現段階では、MDSDRV 系の「1 音の中で音程を変化させる」用途を最小構成で扱うことを目的とします。タムやキック風の下降ピッチ、短い効果音の立ち上がり演出を主な対象としています。

## 1. 定義と選択

- 定義: `@p<num>={initial, speed, step, delay}`
- 選択: `@p<num>`
- 解除: `@p0`

例:

```mml
@p1={3000,1,-150,0}
A @p1 o2 c4
```

## 2. 各要素の意味

- `initial`
  ノート開始時点のピッチオフセットです
- `speed`
  何 tick ごとに `step` を加算するかです
- `step`
  1 回ごとの変化量です。負の値で下降、正の値で上昇します
- `delay`
  変化開始まで待つ tick 数です

## 3. オフセット単位

現行実装では、`100 = 半音 1 つ分` として扱います。

- `100` = +1 semitone
- `-100` = -1 semitone
- `1200` = +1 octave

つまり `@p1={300,1,-25,0}` なら、開始時に 3 半音上から始まり、毎 tick 0.25 半音ずつ下がっていくイメージです。

## 4. 挙動

- `@p` はノートの Key-On ごとに先頭状態へ初期化されます
- `delay` 中は `initial` の位置を維持します
- `delay` 終了後、`speed` tick ごとに `step` を加算します
- 現段階では終端値やループは持ちません
- ノートオフでピッチ変化も停止します

## 5. 用途例

### タム風の下降

```mml
@p1={3000,1,-150,0}
@E1={1,0,2,4,6,8,10,12,14}
A @p1 @E1 o2 c4
```

### 立ち上がりだけ少し上げる

```mml
@p2={200,2,-50,0}
A @p2 o5 l8 c e g
```
